Scoring notes:
A (Canonical) — Perfect fidelity and distinction but outline-only is hard to read at canvas scale, especially when zoomed out. Complex to implement (custom shapes per type).
B (Current) — Easy (already done) but icons are generic (lucide:boxes for a system?), internal/external only distinguished by shade of grey/blue, unicode dropdown icons inconsistent with canvas.
C (FA5) — Marginal improvement over B with slightly better icon semantics, but adds ~400KB dependency for 9 icons. Still uses non-canonical filled-box style.
D (Structurizr) — Clean, from Simon Brown's tool. No icons removes ambiguity. But type text takes space and types look similar without colour distinction.
E (Hybrid) — Best of both worlds: canonical green/blue/red colour coding makes internal/external/person instantly recognisable. Subtle fills maintain readability. Inline SVG glyphs are lightweight, semantically meaningful, and need no dependency. Consistent across canvas, dropdowns, and element views.
